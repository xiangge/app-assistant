# Security Policy

## Reporting Security Issues

Please do not open a public issue for vulnerabilities or privacy-sensitive reports.

If you find a security issue, contact the maintainer privately first. If no private contact is available, open a minimal public issue that says a security report is available, without including exploit details, credentials, account data, or private screenshots.

## Scope

This project contains local automation scripts. Reports are especially useful when they involve:

- Accidental leakage of credentials, tokens, account data, logs, or screenshots.
- Unsafe default behavior around Android accessibility permission.
- Unexpected actions outside the target app package.
- Commands or dependencies that could run untrusted code.

## User Responsibility

Only run the scripts on devices and accounts you control. Review the code before granting accessibility or floating window permissions.
